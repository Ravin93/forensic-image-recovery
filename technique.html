Pipeline général

// ARCHITECTURE DU SYSTÈME DE BOUT EN BOUT
ENTRÉE

Image JPEG ou PNG uploadée via l'interface web ou l'API REST. Validation du format, de la taille et des magic bytes avant traitement.

TRAITEMENT

Pipeline séquentiel : corruption → détection masque → 11 stratégies de reconstruction testées en parallèle → sélection automatique du meilleur candidat par score.

SORTIE

3 images (original / corrompu / reconstruit), score 0–100, stratégie sélectionnée, tableau complet des candidats, rapport JSON et PDF téléchargeable.

Dégradations disponibles

// 15 TYPES DE CORRUPTION RÉALISTES IMPLÉMENTÉS
Rayures
apply_scratch_lines()
Traits fins ou épais tracés en diagonale simulant des égratignures physiques sur un support photographique.
count, thickness, angle_range, color, seed
→ Simulation de négatifs rayés, supports optiques endommagés
Carré supprimé
apply_large_deleted_square()
Suppression d'un grand carré centré couvrant une fraction significative de l'image. Simule une perte de secteur disque.
size_ratio, fill_mode, seed
→ Perte de cluster, secteur défaillant
Barres multiples
apply_multiple_bars()
N barres horizontales ou verticales réparties aléatoirement. Simule des erreurs de transmission ou de lecture linéaire.
count, orientation, thickness, fill_mode, seed
→ Erreurs de transmission, artefacts d'encodage vidéo
Trous aléatoires
apply_random_holes()
N zones rectangulaires de tailles variables dispersées aléatoirement dans l'image.
count, min_size, max_size, fill_mode, seed
→ Corruption aléatoire de clusters, bad blocks
Bruit local
apply_local_noise()
Bruit gaussien ou sel-et-poivre appliqué sur une zone définie. Simule une dégradation capteur ou une interférence électromagnétique.
x, y, width, height, sigma, noise_type, seed
→ Dégradation capteur, interférence EM localisée
Zone supprimée
apply_zone_deletion()
Remplacement d'une zone rectangulaire par du noir, blanc, gris ou bruit. Masque précis retourné.
x, y, width, height, fill_mode, seed
→ Overwrite de zone mémoire, corruption ciblée
Masque rectangulaire
apply_rectangle_mask()
Remplacement d'une zone par une valeur fixe (0=noir, 255=blanc). Corruption déterministe simple.
x, y, width, height, fill_value
→ Démo, tests, référence
Barre unique
apply_bar_corruption()
Une ou plusieurs barres avec espacement contrôlé (gap). Plus précis que multiple_bars pour les cas documentés.
orientation, thickness, position, count, gap, fill_mode
→ Ligne de synchronisation perdue, entrelacement
Blocs abandonnés
apply_block_dropout()
Suppression aléatoire de blocs NxN selon un ratio de perte. Simule une compression avec perte de blocs JPEG.
block_size, drop_ratio, fill_mode, seed
→ Perte de paquets réseau, JPEG partiel
Artefacts JPEG
apply_jpeg_block_artifacts()
Simulation d'artefacts de pixellisation par downscale/upscale sur une zone. Effet mosïaque de mauvaise compression.
x, y, width, height, block_size
→ Sur-compression JPEG, artefacts de recompression
Flou local
apply_local_blur()
Flou gaussien appliqué sur une zone rectangulaire. Simule une mise au point incorrecte ou un mouvement sur une portion.
x, y, width, height, kernel_size
→ Flou de bougé, défaut optique localisé
Bruit gaussien
apply_noise()
Ajout de bruit gaussien sur une zone définie avec intensité sigma paramétrable.
x, y, width, height, sigma, seed
→ Dégradation de capteur, numérisation basse qualité
Décalage de région
apply_shift_region()
Déplace une zone de l'image par translation (shift_x, shift_y). L'original laisse une trace noire/blanche.
x, y, width, height, shift_x, shift_y, fill_mode
→ Corruption de métadonnées d'offset, réassemblage incorrect
Combinée
apply_combined_corruption()
Zone noire + halo de bruit autour. Simulation d'une corruption avec effets de bord (write failure + charge résiduelle).
x, y, width, height, fill_value, sigma, seed
→ Défaillance flash NAND, write failure
Mixte
apply_mixed_corruption()
Combinaison de plusieurs dégradations selon une liste d'opérations. Masque cumulé retourné. Le plus réaliste.
seed, operations: [{type, ...params}]
→ Scénario forensique réel multi-dégradation

Algorithmes de reconstruction

// 11 STRATÉGIES TESTÉES AUTOMATIQUEMENT, MEILLEURE SÉLECTIONNÉE
Inpainting r3 / r5 / r7
Reconstruction OpenCV par Navier-Stokes ou Telea selon le rayon de diffusion (3, 5 ou 7 pixels). Propage les textures environnantes dans la zone masquée.
L'algorithme Navier-Stokes résout les équations de diffusion de couleur depuis les bords du masque vers l'intérieur. Telea utilise une estimation rapide basée sur la distance aux bords. Le rayon contrôle la zone d'influence.
→ Optimal pour : zone_deletion, bar, rectangle_mask, scratch_lines
Denoise median
Filtre médian sur l'image entière. Élimine le bruit sel-et-poivre en remplaçant chaque pixel par la médiane de son voisinage.
Préserve mieux les contours que le filtre gaussien. Insensible aux valeurs extrêmes (outliers). Taille de noyau : 5×5 par défaut.
→ Optimal pour : noise, local_noise, salt_pepper
Denoise gaussian
Convolution gaussienne 2D. Lisse le bruit de manière isotrope en pondérant les pixels voisins selon une distribution gaussienne.
Rapide et simple. Tend à flouter légèrement les contours. Efficace sur le bruit gaussien additif.
→ Optimal pour : noise gaussien, combined
Denoise bilatéral
Filtrage bilatéral préservant les contours : lisse les zones homogènes tout en respectant les discontinuités de couleur.
Combine filtre spatial + filtre de range (intensité). Plus lent que gaussian/median mais produit un résultat visuellement supérieur sur les images naturelles.
→ Optimal pour : bruit + préservation des bords
Deblur light / strong
Défloutage par déconvolution. "Light" : 1 passe. "Strong" : 2 passes successives pour les flous intenses. Utilisé sur les zones floues ou les artefacts de compression.
Implémenté par filtre de netteté (unsharp mask ou Laplacian sharpening). Peut amplifier le bruit si appliqué en excès.
→ Optimal pour : local_blur, jpeg_block_artifacts
Block repair
Réparation spécifique aux artefacts de blocs JPEG : détecte et lisse les frontières entre blocs 8×8 pour éliminer l'effet mosïaque.
Applique un filtre de lissage adaptatif aux jointures de blocs. Moins destructif que le débruitage global.
→ Optimal pour : jpeg_block_artifacts, block_dropout
Hybrid denoise + inpaint
Pipeline composé : débruitage médian d'abord, puis inpainting sur le résultat. Combine la réduction de bruit et la reconstruction de zone manquante.
La passe de débruitage rend les bords du masque plus propres, ce qui améliore la qualité de la diffusion inpainting.
→ Optimal pour : mixed, combined, block_dropout
Conservative (baseline)
Retourne l'image corrompue sans modification. Sert de référence pour vérifier que toute reconstruction améliore bien l'image par rapport à la corruption brute.
Toujours inclus dans les candidats. Si le conservative gagne, cela indique que les reconstructions ont dégradé l'image.
→ Baseline : score de l'image corrompue non reconstruite

Scoring et métriques

// ÉVALUATION QUANTITATIVE DE LA QUALITÉ DE RECONSTRUCTION
MODE SUPERVISÉ

Quand l'image originale est disponible. Score le plus fiable.

PSNR
Peak Signal-to-Noise Ratio. Mesure l'écart moyen pixel par pixel en dB. Plus élevé = meilleure reconstruction.
0–100 dB
SSIM
Structural Similarity. Mesure la similarité perceptuelle (luminance, contraste, structure). Proche de 1.0 = quasi-identique.
0–1
Gain PSNR
PSNR reconstruit − PSNR corrompu. Mesure l'amélioration nette apportée par la reconstruction.
dB
Score global
0.60×SSIM + 0.20×PSNR/100 + 0.30×gain_SSIM + 0.20×gain_PSNR. Normalisé 0–100.
0–100
MODE AVEUGLE

Quand l'original est absent. Score heuristique.

Netteté
Variance du Laplacien. Une image bien reconstruite a des contours nets (variance élevée).
35%
Bruit résiduel
Écart vs filtre gaussien léger. Un bruit élevé pénalise le score.
25%
Continuité
Gradient Sobel moyen. Récompense les bords présents mais pas excessifs.
20%
Cohérence
Delta moyen corrompu → reconstruit. Pénalise la sur-correction.
10%
INTERPRÉTATION DU SCORE GLOBAL
≥ 80
EXCELLENT
Reconstruction quasi-parfaite. Zones masquées visuellement indétectables.
60–79
BON
Reconstruction de qualité. Légères traces possibles sur inspection rapprochée.
40–59
MOYEN
Reconstruction partielle. Zone masquée visible mais cohérente avec le reste.
< 40
FAIBLE
Zone non récupérable. Corruption trop étendue ou complexe pour les algorithmes disponibles.

Modes d'exécution

// TROIS NIVEAUX D'ASSISTANCE DISPONIBLES
Assisté
Le masque exact (généré lors de la corruption) est utilisé directement pour la reconstruction. Aucune détection nécessaire. IoU = 1.0. Donne les meilleurs résultats.

Usage : évaluation des algorithmes de reconstruction, démo, soutenance.
Aveugle basique
Le masque est détecté automatiquement par seuillage de luminosité (zones anormalement sombres). Rapide mais imprécis sur les images naturellement sombres.

Usage : cas forensiques simples (fond clair, zone noire évidente).
Aveugle avancé
Fusion de 3 détecteurs : zones sombres + faible variance + ruptures de gradient. Classification automatique du type de corruption. Plus précis sur images complexes.

Usage : cas forensiques réels sans connaissance de la corruption.
PARAMÈTRES API
POST /pipeline/corrupt-and-repair
image : UploadFile (JPEG/PNG)
corruption_type : scratch_lines | zone_deletion | multiple_bars | ...
severity : light | medium | heavy
execution_mode : assisted | blind_basic | blind_advanced
max_attempts : int (2–11)
seed : int | null

Limites du système

// CONTRAINTES CONNUES ET PÉRIMÈTRE D'APPLICATION
  • Détection aveugle sur images sombres La détection automatique par seuillage de luminosité génère des faux positifs sur les images naturellement sombres (couchers de soleil, scènes nocturnes, canaux). Le mode aveugle avancé améliore ce point mais ne l'élimine pas. Recommandation : utiliser le mode assisté pour les évaluations précises.
  • Inpainting sur grandes zones (>15% de l'image) L'inpainting OpenCV (Navier-Stokes / Telea) est efficace sur les petites zones masquées. Au-delà de 15% de la surface, la diffusion produit des artefacts de flou ou des incohérences de texture. Le score chute et la stratégie conservative peut l'emporter.
  • Scoring aveugle = estimation heuristique Sans image originale, les métriques (netteté, bruit, gradient) sont des proxies de qualité et non des mesures objectives. Le classement des candidats peut différer du classement supervisé. Confiance réduite, interprétation prudente.
  • Reconstructions complexes (shift_region, block_dropout) Les corruptions impliquant un déplacement de région ou la suppression aléatoire de blocs sur des textures complexes (tissu, fourrure, mosaïque) peuvent donner des reconstructions sous-optimales car les algorithmes cherchent à combler des zones dont le contenu originel est non-inferrable depuis les bords.
  • Performance et scalabilité Le pipeline complet (11 stratégies) prend 2–5 secondes sur une image 1080p sur un Mac M-series. Sur images haute résolution (>4K) ou sans accélération matérielle, le temps peut dépasser 30 secondes. Pas de traitement par lot (batch) en l'état actuel.
  • Pas de deep learning / modèle neuronal Toutes les reconstructions utilisent des algorithmes classiques (OpenCV, filtres, convolutions). Les approches deep learning (Stable Diffusion inpainting, LaMa) donneraient de meilleurs résultats sur les grandes zones mais nécessitent GPU et poids de modèle.

Modules forensics avancés

// 12 MODULES DE RECHERCHE — SPRINT K1 À K4
K8 — Stable
PatchMatch Inpainting
app/modules/reconstruction/patchmatch.py
Algorithme exemplar-based inspiré Barnes 2009. Remplissage progressif par patches similaires (onion-peel). Meilleur que l'inpainting OpenCV sur les grandes zones et textures répétitives. Sans GPU.
patchmatch_inpaint(image, mask, patch_size=7, iterations=5)
→ Stratégies : patchmatch_p7_i5 / p9_i5 / p11_i7
K12 — Stable
Analyse EXIF Forensique
app/modules/metadata/exif_analyzer.py
Extraction complète des métadonnées EXIF. Détection de logiciels de retouche (Photoshop, GIMP, Stable Diffusion), incohérences de dates, GPS, orientation suspecte. Score de suspicion 0–1.
extract_exif(path) → dict
analyze_exif_consistency(path) → {suspicious, flags, score}
K14 — Stable
Hash Perceptuel
app/modules/similarity/perceptual_hash.py
dHash (difference hash) et pHash (DCT perceptuel) pour comparer des images même après compression, recadrage ou modification légère. Distance de Hamming. Sans dépendance externe.
compute_dhash(path) / compute_phash(path)
compare_images_perceptual(a, b) → {distance, similarity, label}
K17 — Stable
Détection Copy-Move
app/modules/tampering/copy_move_detector.py
Détecte les zones copiées-collées par feature matching ORB + BFMatcher Hamming. Génère un masque des zones suspectes et un score de confiance. Fonctionne sans GPU ni SIFT payant.
detect_copy_move(path, min_matches=10)
→ {suspicious, strong_matches, regions, mask_path, confidence}
K11 — Stable
OCR sur zones reconstruites
app/modules/ocr/text_detector.py
Extraction de texte avant/après reconstruction. Utilise Tesseract si disponible, EasyOCR sinon. Calcule un gain OCR (text_gain). Ne crash jamais si dépendances absentes.
run_ocr(path) → {text, confidence, backend}
compare_ocr_before_after(before, after) → {text_gain, words_before, words_after}
K19 — Stable
Rapport Forensique Légal
app/modules/reporting/legal_report.py
Enrichit tout rapport avec : chain of custody (SHA-256 à chaque étape), table des méthodes, disclaimer juridique, note légale prudente, certifications. forensic_mode : forensic_safe ou generative.
generate_legal_report(report) → JSON enrichi
build_chain_of_custody(orig, corr, recon, id) → [{step, sha256}]
K13 — Stable
Stéganographie LSB
app/modules/steganalysis/lsb_analyzer.py
Analyse la distribution des bits de poids faible (LSB) par canal RGB. Entropie de Shannon, ratio 0/1. Score d'anomalie 0–1. Niveau de suspicion : faible / moyenne / forte.
analyze_lsb_distribution(path) → {channels, mean_entropy}
detect_lsb_anomaly(path) → {anomaly_score, suspicious, level}
K18 — Stable
Bruit Résiduel / PRNU
app/modules/noise/residual_noise.py
Extraction du bruit capteur par filtre bilatéral. Analyse de cohérence par blocs (détection zones retouchées à >2σ). Heatmap JET. Corrélation PRNU pour identifier même appareil.
compute_noise_consistency(path) → {consistency, regions, heatmap}
compare_prnu(a, b) → {correlation, same_device}
K15 — Stable
Clustering de Fragments
app/modules/fragments/fragment_clustering.py
Regroupe les fragments visuels par DBSCAN sur 107 features (histogramme couleur + stats + entropie + variance). Fallback implémentation maison si sklearn absent.
cluster_fragments_dbscan(paths, eps=0.3)
→ {clusters, noise, n_clusters, coherence_score}
K10 — Optionnel
Chargement RAW
app/modules/raw/raw_loader.py
Chargement DNG, CR2, NEF, ARW via rawpy. Conversion en PNG RGB pour le pipeline. Extraction métadonnées (ISO, exposition, balance blancs). Module 100% optionnel sans crash.
load_raw_as_rgb(path) → {image, width, height}
save_raw_as_png(path) → PNG prêt pour le pipeline
K9 — Expérimental
LaMa Deep Inpainting
app/modules/ai/lama_adapter.py
Adaptateur LaMa (Large Mask inpainting). Active si LAMA_ENABLED=true + torch + modèle présent. Résultat marqué GÉNÉRATIF / NON PROBANT. Jamais en prod sans validation humaine.
run_lama_inpainting(image, mask)
→ {forensic_mode: "generative", warning: "NON PROBANT"}
K16 — Stable
Timeline Forensique
app/modules/timeline/timeline_builder.py
Collecte les événements d'une analyse avec timestamp ISO et elapsed_s. Reconstruit rétrospectivement depuis un rapport JSON. Génère une section HTML pour les rapports.
ForensicTimeline.record(event, details)
build_timeline_from_report(report) → {timeline, n_events, total_s}

Reconstruction avec masque dessiné manuellement (éditeur de masque)

max_attempts: int (1–11)

Compare masque utilisateur vs masque détecté automatiquement
image, user_mask: UploadFile
detection_mode: basic | advanced
max_attempts: int
